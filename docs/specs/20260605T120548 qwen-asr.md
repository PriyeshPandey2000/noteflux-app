# Qwen3-ASR Local Integration

## Goal
Add Qwen3-ASR as local on-device transcription provider. Apple Silicon only. No cloud, no Python, no API key.

## Architecture
Same sidecar pattern as whisperkit branch: build Swift CLI binary → bundle in Tauri → invoke per recording → stream transcript via stdout events.

Key difference from whisperkit: no persistent HTTP server. Invoke CLI per recording, get text back. Simpler UX.

## Approach

### Swift CLI binary (`qwen3-asr-cli`)
- Depends on `speech-swift` Swift package (github.com/soniqo/speech-swift)
- Accepts: `--model <path> --audio <path>` args
- Downloads/caches model to `~/.whispering/models/qwen3-asr/` on first run
- Outputs transcript to stdout
- Streams partials line-by-line for streaming UX (optional)

### Tauri sidecar
- Bundle binary as `qwen3-asr-cli-aarch64-apple-darwin` in `src-tauri/binaries/`
- Add `invoke_qwen3_asr` Rust command in `lib.rs`
- Emit `qwen3-asr://transcript` events for partial/final results

### Frontend service
- `src/lib/services/transcription/qwen3-asr.ts` — mirrors groq.ts pattern
- No API key needed, model path from settings
- Model: `qwen3-asr-0.6b` (4-bit quant, ~400MB)

### Settings
- Add `transcription.qwen3asr.modelPath` setting
- Add `QwenASR` to `TRANSCRIPTION_SERVICE_IDS`
- Add download card component (mirrors `WhisperKitDownloadCard.svelte`)
